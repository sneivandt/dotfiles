# First-Time UI Style

Use this skill when choosing a visual style for a brand-new UI with no established style.

If an existing UI style is present, fit that style instead.

Do not use this skill to override an existing UI, design system, component library, style guide, CSS variables or design tokens, screenshots, or established visual patterns.

## Default Direction

- Dark theme by default.
- Matte colors, not glossy, neon, glassy, or highly saturated.
- Calm, restrained, modern visual direction.
- Charcoal/slate/graphite/ink/muted-neutral backgrounds.
- Avoid pure black as the main background unless already used by the product.
- Muted low-to-medium saturation accents (for example muted blue, desaturated cyan, soft violet, muted green, warm amber).
- Prefer subtle accents over large areas of strong color.
- Avoid loud gradients by default; if gradients are used, keep them subtle and tonal.
- Clean, spacious layouts with strong alignment and consistent spacing.
- Subtle borders/dividers, minimal visual noise, and restrained shadows.
- Modern readable sans-serif typography, restrained font weights, and clear hierarchy.
- Calm solid buttons, quiet secondary actions, and clear but non-harsh focus states.
- Sparse, subtle motion only when it clarifies state changes.
- Overall feel: practical, polished, developer-tool-like, calm, modern, and slightly premium rather than playful or flashy.
